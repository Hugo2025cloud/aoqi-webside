## 分步部署指导

### 1. 在GitHub上创建仓库

#### 步骤：
1. 打开浏览器，访问 [GitHub](https://github.com/)
2. 登录您的GitHub账号
3. 点击右上角的 "+" 图标，选择 "New repository"
4. 输入仓库名称：`aoqi-webside`（或您喜欢的名称）
5. 选择仓库类型：公开（Public）或私有（Private）
6. **重要**：不要勾选 "Initialize this repository with a README"
7. 点击 "Create repository"

### 2. 添加远程仓库

在终端中执行以下命令（请替换为您的实际信息）：

```bash
# 将 yourusername 替换为您的GitHub用户名
git remote add origin https://github.com/yourusername/aoqi-webside.git
```

### 3. 验证远程仓库配置

执行以下命令确认远程仓库已正确添加：

```bash
git remote -v
```

预期输出应该类似：
```
origin  https://github.com/yourusername/aoqi-webside.git (fetch)
origin  https://github.com/yourusername/aoqi-webside.git (push)
```

### 4. 推送代码到GitHub

执行以下命令将本地代码推送到GitHub：

```bash
git push -u origin master
```

#### 可能的错误及解决方案：

- **错误1：权限问题**
  ```
fatal: Authentication failed for 'https://github.com/yourusername/aoqi-webside.git/'
  ```
  解决方案：
  - 确保您的GitHub用户名和密码正确
  - 如果启用了2FA，需要使用个人访问令牌（PAT）代替密码

- **错误2：网络问题**
  ```
fatal: unable to access 'https://github.com/yourusername/aoqi-webside.git/': Failed to connect to github.com port 443: Timed out
  ```
  解决方案：
  - 检查网络连接
  - 尝试使用VPN
  - 考虑使用SSH协议替代HTTPS

### 5. 启用GitHub Pages

#### 步骤：
1. 在GitHub仓库页面，点击 "Settings" 标签
2. 在左侧菜单中选择 "Pages"
3. 在 "Source" 部分：
   - 选择 "Branch: master"
   - 选择文件夹："/root"
4. 点击 "Save"

#### 验证GitHub Pages状态：
- 页面会显示 "Your site is published at https://yourusername.github.io/aoqi-webside/"
- 可能需要几分钟时间网站才能生效

## 替代方案

如果GitHub Pages部署遇到困难，可以尝试以下替代方案：

### 本地测试

使用本地服务器测试网站：

#### 方法1：使用Python
```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```
然后在浏览器中访问：`http://localhost:8000`

#### 方法2：使用Node.js
```bash
npx serve .
```
然后在浏览器中访问：`http://localhost:3000`
